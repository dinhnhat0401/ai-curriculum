**Time: ~8 hours | Files: `app.py`, `Dockerfile`, `docker-compose.yml`**

## The Production Checklist

Every item here is something that separates a demo from a product:

```
    [ ] Containerized (Docker)
    [ ] API with proper error handling (FastAPI)
    [ ] Request/response logging (structured JSON)
    [ ] Token usage and cost tracking
    [ ] Rate limiting
    [ ] Input validation
    [ ] Health check endpoint
    [ ] CI/CD pipeline (GitHub Actions)
    [ ] Documentation (API docs auto-generated by FastAPI)
```

## API Design for AI Services

```
    POST /api/chat          → RAG-powered conversational endpoint
    POST /api/query         → Single question, single answer
    GET  /api/health        → Service status + model availability
```

Key principles:
- **Always return structured responses** (JSON with consistent shape)
- **Include metadata** (latency, token count, sources used, model name)
- **Stream when possible** (Server-Sent Events for token-by-token output)
- **Fail gracefully** (return error messages, not stack traces)

## Monitoring: What to Track

| Metric | Why |
|--------|-----|
| P50/P95/P99 latency | Know your performance distribution |
| Error rate | Catch issues before users report them |
| Token usage per request | Track costs in real time |
| Cost per query | Know your unit economics |
| Request volume | Capacity planning |
| Cache hit rate | Measure optimization effectiveness |

## Guardrails

- **Prompt injection detection**: Check if user input tries to override system prompt
- **PII detection**: Don't log or store personal information
- **Output validation**: Check response format before sending to client
- **Token limits**: Cap input and output to prevent cost explosions
- **Rate limiting**: Protect against abuse and runaway costs
